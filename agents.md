## 1. Maximum Efficiency
- **Algorithmic Optimization:** Always choose efficient data structures and algorithms for the task.
- **Resource Management:** Minimize memory overhead and redundant computations.

## 2. Functional Modularity
- **Short Functions:** Functions should focus on a single responsibility (SRP). Aim for less than 100 lines of code per function.

- **Minimal code:** Use generally as few files and lines of code as possible

## 3. Semantic Clarity
- **Meaningful Variables:** Never use ambiguous names (e.g., `data`, `val`, `temp`). Use descriptive names that reveal intent (e.g., `user_transaction_history`, `is_processing_complete`).
- **Self-Documenting Code:** Write code that is readable without excessive comments. Only comment on "why" something is done, not "what" is being done.

## 4. Automated Testing
- **Test-Driven Development:** Write unit tests for every logical unit created.
- **Framework Preference:** Use `pytest` for Python projects or standard testing libraries appropriate for the language.
- **Coverage:** Ensure edge cases (empty inputs, null values, maximum bounds) are covered in the test suite.

## 5. Implementation Strategy
- When asked to implement a feature, first outline the architectural approach.
- Provide the implementation in concise blocks.
- Immediately follow the implementation with a simple test (again minimal code).

def get_even_squares(numbers: list[int]) -> list[int]:
    """Calculates squares of even numbers using list comprehension for efficiency."""
    return [n * n for n in numbers if n % 2 == 0]

def test_get_even_squares():
    assert get_even_squares([1, 2, 3, 4]) == [4, 16]
    assert get_even_squares([]) == []
    assert get_even_squares([1, 3, 5]) == []
